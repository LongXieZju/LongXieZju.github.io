如何不使用sudo解决Hexo安装时的权限问题

Posted on 2018-04-16 | In Hexo | Comments: | Views

前言

在使用Hexo搭建个人博客时需要先下载安装Node.js，然后才能使用npm install -g hexo-cli命令安装Hexo，但直接使用该命令会报权限错误(permission denied)：

从输出可看出，原因是当前用户无权限对文件夹/usr/local/lib进行写操作。许多人此时会在命令前面加sudo来解决，网上搜到的答案也大多是这种，还有一种是使用chown -R命令改变文件夹所有者使当前用户具备权限。这两种方法虽然能解决问题，但不要这么做，使用sudo会允许命令执行任何操作，这么做非常危险，更改除home文件夹外其他文件夹的所有者也会造成潜在问题。所以不要轻易使用sudo，安装任何模块都应该安装在自己的home文件夹下或者自己创建的虚拟环境下。

下面介绍如何在不使用sudo命令和不更改文件夹所有者的基础上解决安装Hexo时的权限问题，有两种解决方法:

1、使用版本管理工具NVM重新安装npm
2、手动更改npm的默认文件夹（不适用于Windows）

使用NVM重新安装npm

该方法是最好的避免安装权限问题的方法，且不需要重新安装npm或Node.js就可以使用NVM管理，具体做法看这里，Windows／Mac OS／Linux不同系统都有详细介绍。
